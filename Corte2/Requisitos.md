#  Requerimientos del Sistema De Gestión para Restaurantes – RestManage

## Requerimientos Funcionales:

1. El sistema debe permitir el inicio de sesión de usuarios mediante usuario y contraseña.

2. El sistema debe permitir la creación de pedidos ingresando productos y cantidades.

3. El sistema debe calcular automáticamente el total del pedido.

4. El sistema debe permitir confirmar y registrar pedidos.

5. El sistema debe permitir ver los reportes de ventas diarias.

6. El sistema debe permitir generar facturas después de realizar un pedido.

7. El sistema debe registrar las ventas realizadas.

8. El sistema debe permitir seleccionar el método de pago.

9. El sistema debe permitir la gestión de productos (crear, editar y eliminar o dessactivar los productos).

10. El sistema debe permitir la gestión de usuarios (crear, editar y asignar roles).
  

---

## Requerimientos No Funcionales

1. El sistema debe ser fácil de usar y tener una interfaz intuitiva.

2. El sistema debe garantizar la seguridad de la información con autenticación de usuarios.

3. El sistema debe responder de manera rápida a las solicitudes del usuario.

4. El sistema debe estar disponible durante el horario de operación del restaurante.

5. El sistema debe permitir el crecimiento en número de usuarios y datos osea debe tener escalabilidad.
